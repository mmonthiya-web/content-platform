使用说明

欢迎使用内容平台

小红书内容运营工作台，帮你把爆款分析、文案创作、内容排期整合在一个地方。 这份说明会告诉你怎么从零开始用起来。

🚀 立即开始创作 查看首页
快速上手

三步开始使用

第一次使用，按这个顺序来，10 分钟内可以生成第一篇文案。

1

设置 Brand Context

进入创作页，填写你的账号定位、受众信息、语气偏好。 这是告诉 AI「你是谁」的步骤，只需要做一次，之后每次生成文案都会基于这份设定。 填完四步后点「生成 Brand Context」，AI 会自动整理成一份说明。

2

生成第一篇文案

Brand Context 设置好后，在创作页输入主题（比如「果酸水使用教程」）， 选择语气风格，点击「生成文案」。AI 会输出标题、正文、标签和写作思路。 可以切换不同语气版本对比，找到最合适的。

3

保存草稿并安排排期

满意的文案点「保存到草稿库」，进入草稿库可以继续编辑、上传封面图、添加标签。 编辑好之后在排期日历选择发布日期和时间，安排好发布计划。

需要 API Key：创作页的 AI 功能（Brand Context 生成和文案生成）需要 Anthropic API Key。 申请地址：console.anthropic.com。 其他功能（素材库、草稿库、排期日历）不需要 API 也可以正常使用。
功能介绍

五个核心模块

每个模块对应内容创作流程的一个阶段，从找选题到发布全程覆盖。

🏠

首页 · 今日工作台

查看内容流水线状态（灵感库 → 草稿 → 排期 → 发布）， 系统根据当前数据自动生成今日待办，告诉你今天该做什么。

→ 进入首页
📚

素材库 · 内容管理

存放从爆款分析中提取的灵感、Hook、标题、CTA、高频问题。 点击「导入分析结果」将 Claude 的分析内容存入库中。 支持按类型筛选、搜索、查看来源追溯。

→ 进入素材库
✍️

创作 · AI 文案生成

设置 Brand Context 后，输入主题即可生成符合小红书风格的完整文案。 支持多种语气版本对比，可引用素材库中的 Hook 和标题。

→ 进入创作页
📝

草稿库 · 内容管理

管理所有未发布的内容。支持编辑标题、正文、标签、分类， 上传封面图片，查看完成度检查（标题/正文/标签/排期是否齐全）。

→ 进入草稿库
📅

排期日历 · 发布计划

月历和周历两种视图，查看和管理发布计划。 可以把草稿库中的内容拖入日历安排发布时间， 支持改期、取消排期操作。

→ 进入排期日历
使用技巧

如何使用素材库

素材库是整个工作流的起点，内容越丰富，AI 生成的文案质量越高。

1

找到小红书爆款帖子

在小红书上找到同领域点赞收藏高的帖子，复制帖子内容和链接。

2

用 Claude 分析爆款结构

打开 claude.ai， 把帖子内容发给 Claude，要求它分析：标题套路、Hook 类型、情绪触发、文案公式、CTA 策略。 让 Claude 以 JSON 格式输出结果，方便导入。

3

导入到素材库

在素材库页面点击「导入分析结果」，粘贴 Claude 返回的内容。 系统会自动提取 Hook、标题、CTA 等素材分类存入对应模块。

建议：先积累 3-5 篇爆款拆解，再开始用 AI 生成文案。素材越多，生成的内容越贴合你的领域和风格。

常见问题

FAQ

怎么获取 Anthropic API Key？
访问 console.anthropic.com， 注册账号后进入 API Keys 页面创建新的 Key。 Key 格式为 sk-ant-...，复制后在创作页的 Brand Context 设置里使用。 注意：API Key 只会保存在你自己的浏览器里，不会上传到任何服务器。
数据存在哪里？刷新页面会不会丢失？
所有数据（Brand Context、自定义语气标签、生成历史）存在浏览器的 localStorage 里。 刷新页面不会丢失，但清除浏览器数据或换浏览器会丢失。 草稿、素材、排期数据目前是演示数据，后续会接入真实的数据持久化方案。
Brand Context 是什么？为什么要填？
Brand Context 是告诉 AI「你是谁」的一份说明，包括你的账号定位、目标受众、语气风格、禁用词等。 每次生成文案时，AI 都会参考这份说明，确保生成的内容符合你的风格，而不是通用模板。 填得越详细，生成质量越高。只需要填写一次，之后会自动沿用。
如何修改这份使用说明的内容？
直接在 GitHub 上编辑 guide.html 文件。 所有文字内容都写在 HTML 标签之间，不需要懂代码，找到对应的文字直接修改就好。 保存后 GitHub Pages 会在 1-2 分钟内自动更新。
可以添加新的使用说明章节吗？
可以。在 guide.html 里复制一个 <div class="section">...</div> 块， 修改里面的标题和内容即可。也可以用现有的组件： step-item（步骤）、fcard（功能卡片）、 tip info/warn/good（提示框）、faq-item（折叠问答）。
手机上能用吗？
目前各页面在手机上可以访问，但布局针对桌面端优化，部分功能在手机上体验较差。 手机端适配正在开发中，后续会推出移动端优化版本。
